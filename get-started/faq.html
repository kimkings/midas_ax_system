FAQ

MIDAS AX System에 대해 자주 묻는 질문을 모았습니다.

AI 에이전트가 제품 안에서 사용자와 만나는 방식을 표준화한 피그마 기반 UI 템플릿 가이드입니다. 다양한 AX 경험을 에이전트 경험의 범위와 밀도에 따라 Agentic · Contextual · Background · Builder로 나누어 제공합니다.

새로운 AX 기능을 설계하는 제품 담당자와 디자이너를 위한 공통 자산입니다. 기능의 목적에 맞는 AX UI 유형을 빠르게 판단하고, 일관된 사용자 경험을 유지하면서 화면을 구성할 수 있습니다.

AI 에이전트가 사용자와 만나는 화면을 새로 설계할 때 사용합니다. 직접 대화형(Agentic), 화면 맥락 보조형(Contextual), 백그라운드 처리형(Background), 에이전트·워크플로우 구성형(Builder) 중 기능 성격에 맞는 템플릿을 선택합니다.

피그마 파일에 P1 Agent Kit을 디자인 시스템으로 연결한 뒤, 기능에 맞는 템플릿을 Workspace로 드래그 앤 드롭해 사용합니다. 내부 요소를 swap하거나 옵션을 바꿔 필요한 구성으로 커스터마이징할 수 있습니다.

아니요. 이 가이드는 완성된 정답이 아니라 AX 기능 설계를 시작하기 위한 공통 기반입니다. 실제 기능의 성격, 데이터 구조, 사용자 권한, 적용 범위에 따라 템플릿을 조합하거나 일부 요소를 조정해 사용할 수 있습니다.

2026년 5~6월 TGC 멤버들이 AX 기능 설계·적용 과정에서 반복적으로 등장하는 UI 구조를 정리해 만들었습니다. 실제 제품 적용 사례와 신규 요구사항을 반영해 지속적으로 업데이트됩니다.
