Overview

Dedicated Agent Workspace는 사용자가 특정 작업이나 산출물을 완성하기 위해 별도의 AI 작업공간에 진입하는 UI 패턴입니다. 사용자는 단순히 AI에게 질문하고 답변을 받는 것이 아니라, 목표·조건·참조 정보 등을 바탕으로 AI와 함께 문서, 계획, 분석 결과, 리포트, 코드, 디자인 산출물 등을 만들어갑니다. 대화는 작업을 지시하고 조정하는 수단이며, 화면의 중심은 최종 결과물이나 작업 진행 과정에 있습니다. 이 패턴은 AI를 제품 전역에서 호출하는 공통 인터페이스로 제공하기보다, 특정 업무를 완성하기 위한 전용 인터페이스로 제공할 때 적합합니다.

이 분류를 선택하는 경우

선택 조건설명
특정 작업이나 산출물을 완성해야 한다사용자가 단순 답변이 아니라 문서, 계획, 분석 결과, 리포트처럼 완성된 결과물을 만들어야 하는 경우
작업 시간이 길거나 반복 수정이 필요하다한 번의 질문과 답변으로 끝나지 않고, 조건 수정, 재생성, 편집, 검토가 반복되는 경우
결과물을 별도 영역에서 보고 다뤄야 한다AI 응답이 메시지 안에 머무르기보다, 캔버스나 작업 영역에서 확인·편집·저장되어야 하는 경우
작업 맥락을 유지해야 한다사용자가 여러 단계에 걸쳐 같은 목표, 참조 자료, 이전 결과를 유지하며 작업해야 하는 경우
대화보다 작업공간이 중심이어야 한다AI와의 대화는 보조 수단이고, 사용자의 주 관심은 결과물 완성이나 작업 진행에 있는 경우

이 템플릿을 사용하지 않는 경우

선택하지 않는 경우대신 고려할 패턴
서비스 어디서든 가볍게 AI에게 질문하고 답변을 받으면 충분하다Global Chat Assistant
현재 화면을 유지한 채 측면에서 AI가 보조하면 충분하다Contextual Agent Panel
특정 텍스트나 필드 주변에서 짧게 AI 제안을 받으면 된다Embedded AI Elements
AI 작업이 백그라운드에서 실행되고 사용자는 상태만 확인하면 된다Background Task Monitor

Key Characteristics

구분내용
진입 방식작업 시작 버튼, 템플릿 선택, 특정 업무 플로우에서 Workspace 진입
입력 방식목표, 조건, 요구사항, 참조 데이터, 기존 산출물, 추가 지시
응답 방식문서 초안, 계획안, 분석 리포트, 코드, 구조화된 결과물, 작업 단계
컨텍스트 활용작업 목표, 이전 대화, 참조 자료, 사용자 편집 내용, 버전 히스토리
사용자 흐름작업 시작 → 조건 입력 → 결과 생성 → 편집·수정 → 저장·내보내기
주요 가치AI와 함께 특정 산출물을 완성하는 집중형 작업 경험 제공

주요 예시 레퍼런스

실제 제품에서 Dedicated Agent Workspace 패턴이 어떻게 구현되는지 보여주는 대표 사례입니다.

Sana.ai

사용자가 채팅으로 업무 요청을 입력하면, AI가 연결된 지식 소스와 업무 데이터를 바탕으로 문서나 리포트 형태의 산출물을 생성합니다. 생성된 결과는 단순 메시지 응답에 머무르지 않고, 별도 문서 영역에서 확인·편집·공유할 수 있습니다. 특히 좌측의 대화 영역과 우측의 문서 캔버스가 함께 배치된 구조는 Split View형 Dedicated Agent Workspace의 좋은 참고 사례입니다. Sana의 Docs는 AI 생성 결과를 구조화되고 공유 가능한 콘텐츠로 전환하는 작업공간으로 설명됩니다.

Canva Magic Studio

사용자가 프롬프트와 이미지 같은 입력 자료를 제공하면, AI가 디자인 초안을 생성하고 이를 Canva 편집 화면에서 바로 수정할 수 있습니다. 결과물이 채팅 메시지가 아니라 실제 디자인 캔버스 위의 편집 가능한 산출물로 전환된다는 점에서 Canvas-first Workspace의 대표 사례로 볼 수 있습니다. Canva는 Magic Studio를 Canva 내부에 통합된 AI 기능 묶음으로 소개하며, 브레인스토밍부터 최종 결과물 완성까지 돕는다고 설명합니다.

Lovable

사용자가 자연어로 앱이나 웹사이트 요구사항을 입력하면, AI가 실제 UI와 코드 기반 결과물을 생성하고 사용자가 이를 계속 수정·복원·배포할 수 있는 작업공간을 제공합니다. 좌측에는 요청과 변경 이력이, 우측에는 생성된 앱 화면이 배치되어 있어 대화와 결과물을 동시에 다루는 구조를 참고하기 좋습니다. Lovable은 AI 기반 플랫폼으로 앱, 웹사이트, 디지털 제품을 빠르게 만들 수 있다고 소개합니다.

Basic Layout Structure

Dedicated Agent Workspace는 특정 작업을 완성하기 위한 독립적인 작업 화면 구조를 가집니다. 하위 분기에 따라 Split View 또는 Canvas-first Workspace로 나뉘지만, 기본적으로는 Workspace Header, Instruction / Chat Area, Work Canvas, Action Area로 구성됩니다.

레이아웃 원칙

원칙설명
산출물 중심성화면의 중심은 AI 대화가 아니라 사용자가 완성하려는 결과물이어야 합니다.
작업 지속성사용자가 작업을 중단했다가 돌아와도 목표, 결과물, 히스토리가 유지되어야 합니다.
편집 가능성AI가 만든 결과물은 읽기 전용이 아니라 수정, 재생성, 저장, 내보내기가 가능해야 합니다.
맥락 보존작업 목표, 참조 자료, 이전 대화, 사용자 수정 내용이 작업공간 안에서 유지되어야 합니다.

Variants

Dedicated Agent Workspace는 결과물 작업 영역과 대화 영역을 어떻게 배치하는지에 따라 하위 UI가 나뉩니다.

하위 분기한 줄 정의선택 기준
Canvas-first Workspace결과물 작업 영역을 중심에 두고, 대화나 지시는 보조적으로 배치사용자의 주 관심이 대화보다 산출물 편집과 완성에 있을 때
Split View대화 영역과 결과 작업 영역을 나란히 두고 함께 작업사용자가 AI와 대화하며 결과를 동시에 확인·수정해야 할 때

Canvas-first Workspace (Right)

결과물 편집과 완성을 중심으로 하고, AI 대화나 지시는 보조 패널 또는 인라인 액션으로 제공되는 UI입니다.

Canvas-first Workspace (Left)

결과물 편집과 완성을 중심으로 하고, AI 대화나 지시는 보조 패널 또는 인라인 액션으로 제공되는 UI입니다.

Split View

대화 영역과 결과 작업 영역을 함께 배치하여, 사용자가 AI에게 지시하고 결과물을 동시에 확인하는 UI입니다.

Interaction

Trigger - Workspace Entry

사용자가 특정 작업 시작 버튼, 템플릿 선택, 생성 요청, 기존 산출물 편집 버튼 등을 통해 전용 Workspace에 진입합니다.

전역 에이전트를 호출하는 버튼, 내비게이션 요소를 통해 AI Assistant를 호출합니다.

호출된 AI Assistant는 별도의 전용 공간(새 탭, 새 창 등)에 UI가 구성됩니다. UI는 자연어 기반 상호작용을 기반으로 하되, 산출물의 비중이 높은 2개의 레이어 구조입니다.

Input - Goal and Constraints

사용자는 만들고자 하는 결과물의 목표, 조건, 참고 자료, 원하는 형식, 수정 요청을 입력합니다. 필요에 따라 기존 산출물, 문서, 데이터, 화면 맥락이 함께 전달될 수 있습니다.

사용자는 Composer와 상호작용하여 질문, 요청, 명령, 조건 등을 자연어로 입력합니다.

Output - Editable Artifact

AI는 문서, 계획안, 분석 결과, 코드, 표, 리포트처럼 작업 가능한 산출물을 생성하고 Workspace 안에 표시합니다.

사용자의 요청사항에 따라 대화 기반의 응답을 제공합니다.

구조화된 산출물은 캔버스 패널에서 렌더됩니다.

Follow up - Refine and Complete

사용자는 결과물을 직접 편집하거나 AI에게 수정 요청을 하고, 최종 결과를 저장·공유·적용·내보내기 할 수 있습니다.

사용자가 자연어로 후속 질문을 요청할 수 있습니다.

후속 요청에 대한 산출물을 캔버스에서 렌더하며 산출물을 점진적으로 개선합니다. 필요시 액션 버튼을 통해 저장, 공유 등의 추가적인 액션을 수행할 수 있습니다.
